# CH641

RISC-V2A, 48MHz, 16KB Flash, 2KB SRAM.

USB-PD/Type-C, BC1.2, HV DCP.
ISP/ISN, QII.

> **Note**
> CH641 is a riscv32ec core, which is not supported by ofiicial Rust yet.
> ch32-rs team maintains a fork of Rust at <https://github.com/ch32-rs/rust>.
> You can check [Noxim's Blog](https://noxim.xyz/blog/rust-ch32v003/introduction/) for riscv32ec support.

## In `wlink`

CH641 is treated like **CH32V003** for probe traffic: same **flash-programming blob**, **64-byte** USB data packets, and **1024-byte** fast-program chunks (`write_pack_size`).

## Boot area and option bytes (`START_MODE`)

`wlink` enables **boot-area selection** for CH641 the same way as for CH32V003 / CH32V007: **bit 5** of the **USER** option byte selects Code FLASH vs System FLASH after reset.

**`system-boot` without a `.bin`:** same as CH32V003 — halt, read back the first **1024 bytes** from the start of code flash, run one program session with `BootFrom`, and re-write that region unchanged (see [CH32V003.md](CH32V003.md#boot-area-and-option-bytes-start_mode)).
